实验室预约管理系统需求规格说明书 (基于 RuoYi 框架)1. 项目概述本系统旨在通过数字化手段解决高校实验室资源分配不均、预约流程繁琐及冲突频发的问题 。系统采用前后端分离架构，后端基于 Spring Boot，前端基于 Vue.js 与 Element UI，并以 RuoYi (若依) 开源框架作为基础底座 。  2. 角色功能需求2.1 学生用户 (Student)   实验室查询：支持根据日期、实验室名称、所属楼宇或设备关键字检索空闲资源 。  在线预约申请：选定目标时段并填写实验用途提交申请，系统需即时进行冲突预检 。  记录跟踪与撤销：实时查看个人预约单的审批进度，在审核完成前允许撤销申请 。  设备故障报修：发现设备损坏时在线提交报修单 。  2.2 指导教师 (Teacher)   预约审批：对所辖实验室的学生预约申请执行“通过”或“驳回”操作 。  教学占用申请：代表班级申请固定时段的教学占用，该操作具备最高优先级，可自动拦截同时间段的个人预约 。  报修进度查看：掌握实验室内设备的维修状态以调整教学计划 。  2.3 系统管理员 (Admin)   用户与权限管理：基于 RBAC 模型 管理全校师生账号及角色的按钮级权限 。  资源档案维护：管理实验室物理台账，包括编号、容量、实景图及设备清单 。  全局调度与干预：监控全校排期，强制撤销异常预约（如长时间占用却未到场）以释放资源 。  系统参数与公告：维护数据字典并发布实验室规章制度或临时关闭通知 。  3. 数据库逻辑结构设计 (核心表)   3.1 实验室信息表 (sys_lab)   名称类型长度注释labID (PK)int10实验室编号LabNamevarchar50实验室名称buildingvarchar50所在楼宇capacityint4容纳人数statusint2状态 (0:正常, 1:维修)equipmentvarchar255设备清单3.2 预约信息表 (sys_reservation)   名称类型长度注释ReserveID (PK)int10预约编号userIDint10预约人编号labIDint10实验室编号startDatedatetime-开始时间endDatedatetime-结束时间auditStatusint2审核状态 (0:待审, 1:通过, 2:驳回)remarkvarchar255实验备注3.3 设备报修表 (sys_repair)   名称类型长度注释repairID (PK)int10报修单号labIDint10实验室编号userIDint10报修人编号faultDescvarchar255故障描述repairStatusint2维修状态 (0:待处理, 1:维修中, 2:已修复)submitTimedatetime-提交时间3.4 教学占用表 (sys_teaching_occupancy)   名称类型长度注释occupyID (PK)int10占用编号teacherIDint10教师编号labIDint10实验室编号courseNamevarchar100课程名称occupyDatedate-占用日期timeSlotvarchar50使用时段4. 核心业务算法：预约冲突检测   系统创新性地设计了基于时间交集判定模型的算法 ：  输入：目标实验室编号、申请起始时间戳、申请结束时间戳 。  检索：查询数据库中该实验室所有状态为“已通过”或“待审核”的排期记录 。  判定公式：若当前申请区间与已有记录存在交集（即 $Max(Start1, Start2) < Min(End1, End2)$），则判定为冲突 。  处理：若判定冲突，系统触发拦截机制，回滚数据库事务，并向前端抛出红字警告 。
